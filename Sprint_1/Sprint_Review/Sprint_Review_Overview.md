Sprint 1 Review

During the sprint review, we discussed the implementation of each user story and how they turned out via a demo.
We also discussed what went well during this past sprint, what went poorly, and what we could improve upon for the next.

What we feel went well for Sprint 1:
- Implementation of the backend and database was smooth and seamless. All backend components integrated with one another perfectly.
- Deployment went well. No major issues were encountered while deploying our frontend.

What we feel did not go well for Sprint 1:
- Profile creation gave us some trouble. We received 401 status codes and had to troubleshoot.
- Login for existing users is experiencing latency. We have been working on troubleshooting.

What we can improve upon for the next sprint:
- We need to make the create profile and update profile pages mutually exclusive.
- We need to implement Google SSO during the next sprint.
